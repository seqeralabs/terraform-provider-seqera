# Manual Edits to Generated Code

This file tracks every manual edit made to Speakeasy-generated files in this repo.

The edits are preserved across `speakeasy run` regenerations by the `persistentEdits` feature (`.speakeasy/gen.yaml: generation.persistentEdits.enabled: true`), which uses a 3-way merge anchored by `pristine_git_object` refs in `.speakeasy/gen.lock`.

## When to add an entry

Any time you hand-edit a file with the `// Code generated by Speakeasy (https://speakeasy.com). DO NOT EDIT.` header, add it to the table below.

Workflow:

1. Make the change in the generated file.
2. Add or update an entry in the table below — what, where, and why.
3. Run `speakeasy run` and confirm the edit survives.
4. Commit the edit, the changelog entry, and this file together.

## When to remove an entry

- The upstream generator (or an OpenAPI overlay) starts emitting the same code natively → drop the manual edit and remove the entry.
- The surrounding code is restructured enough that Speakeasy emits a merge conflict the next time it regenerates → reconcile by hand, then update the location/note here.

## When to prefer an overlay over a manual edit

Manual edits are a last resort. Reach for an OpenAPI overlay (`overlays/*.yaml`) or a Speakeasy extension (`x-speakeasy-*`) first — those are visible to the generator and survive everything cleanly. Only fall back to a manual edit when no overlay/extension can express what you need.
